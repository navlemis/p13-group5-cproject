2. CORE OPERATIONS
------------------------------------------------------------
Your CMS must support the following commands:

1. OPEN
   - Opens the database file and reads all records.

2. SHOW ALL
   - Displays all current records in memory.

3. INSERT
   - Adds a new record.
   - Rejects duplicate IDs.

4. QUERY
   - Searches for a record by student ID.
   - Displays data if found, otherwise show warning.

5. UPDATE
   - Updates record data for a specific student ID.

6. DELETE
   - Deletes a record by student ID (requires confirmation).

7. SAVE
   - Saves all current records into the database file.

3. ENHANCEMENT FEATURES
------------------------------------------------------------
To achieve a higher grade, include the following:

A. Sorting Features
   - Sort by Student ID (ascending/descending).
   - Sort by Mark (ascending/descending).
   - Example: SHOW ALL SORT BY ID / SHOW ALL SORT BY MARK.

B. Summary Statistics
   - Command: SHOW SUMMARY
   - Display:
     - Total number of students
     - Average mark
     - Highest and lowest marks with student names

C. Unique Feature
   - Design your own additional feature that:
     - Demonstrates C programming techniques.
     - Fits naturally into the CMS.
